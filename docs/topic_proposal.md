## 一、研究问题

**核心问题**：哪些研发资本化特征与交易所问询相关？

### 1.1 问题拆解

本研究将核心问题拆分为三个可检验的子问题：

| 子问题 | 分析维度 | 数据来源 |
|--------|----------|----------|
| **子问题 1**：高资本化率是否更容易与问询相关？ | 资本化率水平 vs 行业偏离度 | 150 份年报 |
| **子问题 2**：资本化率的剧烈变化是否更容易与问询相关？ | 跨期变化幅度（2021→2022→2023） | 150 份年报 |
| **子问题 3**：资本化条件描述模糊是否更容易与问询相关？ | 条件文本模糊度评分 | 150 份年报 + 问询标签 |

### 1.2 为什么选这三个维度

- **资本化率水平**：直接反映公司会计政策选择的激进程度，同行业对比可剔除行业差异
- **跨期变化**：单一年度的高资本化率可能有合理原因，但连续大幅波动更可能反映"利润调节"动机
- **条件模糊度**：资本化需要满足"技术可行性""未来经济利益很可能流入"等条件，条件描述越模糊，人为判断空间越大

## 二、数据范围与研究问题的一致性

### 2.1 数据范围

| 维度 | 设计 | 与研究问题的对应关系 |
|------|------|---------------------|
| 公司池 | 50 家（医药制造 20 + 电子设备 20 + 软件信息 10） | 覆盖研发密集型行业，资本化行为差异显著 |
| 时间范围 | **2021、2022、2023 年** | 支持**跨期趋势分析**（三年面板数据） |
| 年报 PDF | 150 份（50 家 × 3 年） | 主数据源：抽取资本化率、条件描述 |
| 问询函 | 不预设硬数量；先做全样本候选发现 | 验证材料：检验模型预测是否与实际监管关注信号一致 |

### 2.2 "趋势"问题的数据支撑

研究问题中的"变化趋势"由以下数据支撑：

- **单公司纵向**：每家公司的 2021→2022→2023 资本化率变化轨迹，计算年度变化百分点
- **评分模型**：四维度评分中的 `change_score`（跨期变化幅度评分）直接度量趋势异常
- **风险排序**：标记"资本化率连续三年上升"或"单年正向跳变 > 20 个百分点"的公司

> **关键设计决策**：不做"2021–2023 年资本化率趋势"的时间序列回归（样本量不足），而是做**跨期变化幅度**的横截面比较（150 条记录中每条记录携带 `prev_year_rate` 和 `change_pct`）。这保证了研究问题中的"变化"有数据支撑，同时避免了样本量不足的统计问题。

## 三、核心变量定义

### 3.1 验证标签

| 变量 | 定义 | 数据来源 |
|------|------|----------|
| `is_inquired` | 该 company-year 在年报发布后 180 天内是否收到研发资本化相关问询 | 巨潮资讯网前端公开查询 + 问询文本内容判断 |

### 3.2 特征变量

| 变量 | 定义 | 口径 | 计算方式 |
|------|------|------|----------|
| `capitalization_rate` | 资本化率（%） | 优先使用年报披露值，其次 validate 重算值，再用资本化 / (资本化 + 费用化)，最后才用资本化 / 研发投入总额 | 从年报"研发投入"章节抽取 |
| `change_pct` | 资本化率年度变化（百分点） | 本年 − 上年 | 需连续两年数据 |
| `industry_percentile` | 行业百分位 | 本公司资本化率在同行业同年的排名 | 行业优先读 `configs/crawl.yaml` 显式 `industry` 字段，缺失时顺序推断 fallback |
| `fuzziness_score` | 条件模糊度评分 | 当前 MVP 用关键词命中估计资本化条件描述的模糊程度 | 0–1，越高越模糊 |

### 3.3 分组与描述变量

| 变量 | 说明 |
|------|------|
| `industry` | 行业分类（申万一级：医药生物 / 电子 / 计算机） |
| `year` | 年报年份（2021/2022/2023） |
| `rd_expense_total` | 研发支出总额（万元），用于描述规模差异和交叉校验 |

## 四、分析方法

### 4.1 风险评分与数据质量校验

风险评分模型采用可解释的金融规则，将每个 company-year 映射为 `aggressiveness_score`。
核心信号包括：

- **行业偏离度**：本公司资本化率在同行业同年度中的百分位；
- **跨期变化幅度**：资本化率年度变化的 Z 分数（当前用绝对值，同时捕捉突然上升和下降）；
- **条件模糊度**：资本化条件文本中模糊关键词的命中密度；
- **恒等式置信度**：把“资本化 + 费用化 ≈ 总额”的一致性作为最终风险分乘数（折扣），而不是激进信号。

完整公式、变量定义、阈值与启发式参数审计表见 [`docs/methodology.md`](../docs/methodology.md)。

`identity_check_score` 的计算口径为：

```text
diff_ratio = abs(capitalized + expensed - total) / total
identity_check_score = max(0, 1 - diff_ratio / 0.20)
```

也就是说，金额关系越不一致，最终风险分越被折扣。这个设计把“公司风险”和“抽取/口径质量问题”分开，便于课堂解释。当前 validate 阶段先用 5% 阈值拦截严重恒等式不一致记录，因此进入评分的三金额齐全记录通常只会受到轻微折扣；`identity_error` 异常类型基本不会触发。

### 4.2 问询闭环可行性测试

1. **模型预测**：对 150 条记录计算 `aggressiveness_score`，只在有风险分的记录中取前 20% 标记为异常
2. **候选发现**：对全部 150 个 company-year 查询年报发布后 180 天内的问询函、回复函、关注函或监管工作函候选
3. **内容标签**：脚本层先排除回复类公告，再按 Tier-1/Tier-2 关键词剪枝；仅命中泛词时由 LLM 判断函件是否实质针对研发资本化。详见 [`docs/methodology.md` §6.3](../docs/methodology.md#63-问询函相关性-v2)
4. **混淆矩阵**：将 150 条评分记录与问询标签 join 后计算 TP / FP / TN / FN
5. **评估指标**：有效问询样本充足时输出 Precision / Recall / F1；样本不足时输出 Top-K 命中率、候选覆盖率和典型案例分析

### 4.3 当前 MVP 结果

2026-06-15 使用已完成的 extract 输出重跑后段：

```powershell
uv run python src/main.py --from-stage validate --to-stage report
```

| 指标 | 结果 | 解释 |
| --- | ---: | --- |
| 年报样本 | 150 | 覆盖理论样本 50 家 × 3 年 |
| validate | 150/150 passed | 结构化记录全量通过 Schema 与会计校验 |
| 可计算资本化率 | 60 | 可直接进入资本化率描述统计 |
| 有风险分记录 | 83 | 可用于异常排序 |
| 异常样本 | 17 | 在 83 条可评分记录中取前 20% 左右，换算到 150 全样本为 11.33% |
| 问询候选样本 | 10 | 年报后窗口期内有问询/回复/监管函候选 |
| 研发资本化相关问询样本 | 1 | v2 标签：排除回复类公告 + Tier-1 关键词剪枝 + LLM 语义二分类，单独命中泛词不再判相关 |

当前结论不是“模型已经能预测问询函”，而是：年报文本已经能转成结构化财务指标，风险模型能给出可解释的复核清单；问询函闭环已经接上，但需要全文语义标签和人工复核来提高监管解释力。

## 十、风险评估

### 风险 1：问询函数量不足

**描述**：50 家公司 × 3 年中，实际只有 5-10 家公司有研发资本化相关问询函，Precision / Recall / F1 不稳定。

**缓解方案**：

- 扩大公司池到 80 家（增加软件、半导体、新能源等研发密集型行业）。
- 放宽筛选条件：不仅筛选“研发资本化”关键词，还包括“研发费用”“开发支出”“无形资产”等。
- 若仍不足，增加“关注函”和“监管工作函”。
- 样本仍不足时，改用 Top-K 命中率、候选覆盖率和典型案例分析，不强行解释 F1。

### 风险 2：问询函与年报的时间错位

**描述**：问询函通常针对最新一期年报，但公司可能在年中收到问询，此时上一年年报已经发布。

**缓解方案**：通过 `stock_code` + 年报年份关联，并以年报发布日期后 180 天作为主窗口；发布前监管记录只作为背景变量，不进入触发标签。

### 风险 3：MinerU 表格解析质量

**描述**：开发支出附注表格解析不准，导致评分模型输入数据有噪声。

**缓解方案**：

- 设计会计恒等式校验规则。
- 对异常值人工抽查。
- 评分模型中引入“数据可信度”权重。
