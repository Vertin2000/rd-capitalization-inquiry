# 巨潮数据探索记录

## 方向 A：研发资本化风险排序

- **关键词**：研发支出、资本化、费用化、开发支出
- **公告类型**：年度报告（API 编码通常为 `category_ndbg_szsh`；项目配置中对应 `configs/crawl.yaml` 的 `category_rsg` 参数）
- **示例公司/代码**：恒瑞医药（600276）、迈瑞医疗（300760）
- **示例标题**：2023年年度报告
- **PDF 是否可下载**：是
- **可能字段**：研发支出总额、资本化金额、费用化金额、资本化率、开发支出余额
- **金融问题**：哪些资本化特征更容易伴随监管问询？
- **风险点**：
  - MinerU 对复杂表格的解析质量
  - 不同公司披露格式差异大
  - 部分公司研发支出全部费用化（资本化率为0）

## 方向 B：问询函闭环分析

- **关键词**：问询函、关注函、研发、资本化、开发支出
- **公告类型**：问询函/关注函（历史候选分类码曾记为 `category_dshgszz`，当前尚未小样本校准）
- **示例公司/代码**：部分研发密集型公司
- **示例标题**：关于XX公司2023年年度报告的问询函
- **PDF 是否可下载**：是
- **可能字段**：问询日期、问询问题、回复内容、回复充分性
- **金融问题**：交易所问询函是否针对财务异常？回复是否充分？
- **风险点**：
  - 问询函数量可能不足
  - 问询函与年报的时间错位
  - 回复函内容可能模糊

## 初步选择

我们更倾向于：**方向 A + 方向 B 结合**

**理由**：
1. 方向 A（研发资本化）提供了丰富的财务数据，可构建评分模型
2. 方向 B（问询函）提供了外部监管关注信号，可用于描述性验证模型
3. "风险排序 → 问询函可行性测试"的闭环在金融分析上有探索意义（受限于正样本量，定位为可行性测试而非验证）
4. 结合后的任务复杂度满足挑战档 1.1x 的全部维度要求
