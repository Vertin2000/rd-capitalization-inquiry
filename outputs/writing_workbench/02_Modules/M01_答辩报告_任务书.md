# M01 答辩报告任务书

## 定位

手写答辩版 `final_report.md`（根目录）。面向课程教师（Rubric §11.7）和答辩现场同学，5000-8000 字中文。围绕 §11.7 五维 Rubric 重构叙事，做成可独立阅读的答辩文档。

## 必读材料

| 材料 | 路径 | 用途 |
| --- | --- | --- |
| 硬数字真源 | `outputs/loop_evaluation.json` | 所有数字的单一真源 |
| 方法论 | `docs/methodology.md` | 公式、阈值、口径 |
| Demo 脚本 | `demo_script.md` | 中兴通讯证据链 |
| 自动版底稿 | `outputs/final_report_auto.md` | 参考章节结构 |
| 选题方案 | `docs/topic_proposal.md` | 研究问题与差异化 |
| AI 使用声明 | `ai_usage_statement.md` | Vibe Coding 反思引用 |
| 评估报告 | `outputs/reports/eval_report_final.md` | 评估局限引用 |
| Context | `outputs/writing_workbench/00_Context/01_rubric_and_rules.md` | Rubric + 硬数字 + 约束 |

## 最小交付物

`project2/final_report.md`（根目录），包含 15 个章节，硬数字与 `loop_evaluation.json` 一致。

## 不得写入内容

- ❌ "模型预测问询函" → 说"验证了用问询函做外部参照的可行性"
- ❌ "发现违规" / "违规判定" → 说"风险排序" / "复核清单"
- ❌ "精准识别" / "准确预测" → 说"可行性测试" / "信号混合"
- ❌ 自算数字 → 必须从 loop_evaluation.json 逐字抄
- ❌ 声称问询标签为"真值" → 标注为 MVP
