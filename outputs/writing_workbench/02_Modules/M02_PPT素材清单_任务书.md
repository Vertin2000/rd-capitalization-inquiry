# M02 PPT 素材清单任务书

## 定位

为 Phase 4 `/pptx` 技能提供结构化输入。每页一节，含标题、要点、数据来源、讲者备注、Rubric/检查表映射。目标是 14 页左右的答辩 slides。

## 必读材料

| 材料 | 路径 | 用途 |
| --- | --- | --- |
| 硬数字真源 | `outputs/loop_evaluation.json` | 数字引用 |
| 展示检查表 | `docs/presentation_checklist.md` | §12.20 逐项覆盖 |
| Demo 脚本 | `demo_script.md` | 证据链 Demo 页 |
| 方法论 | `docs/methodology.md` | 公式用于模型页 |
| Context | `outputs/writing_workbench/00_Context/01_rubric_and_rules.md` | Rubric 映射 + 检查表映射 |
| M01 草稿 | `outputs/writing_workbench/04_Drafts/M01_v1.md` 或 `project2/final_report.md` | 已稳定的报告叙事 |

## 最小交付物

`outputs/ppt_outline.md`，14 页节，每页含：标题 + 要点(3-5条) + 数据/图表来源 + 讲者备注 + Rubric/检查表维度标注。

## 页序

1. 封面
2. 金融问题（Rubric ①）
3. 数据来源与难度档
4. Schema 设计
5. MinerU 解析 + Section 检查
6. Workflow 全图
7. 风险评分模型四维（Rubric ②）
8. 结果与 Top-K（Rubric ③）
9. Demo 证据链（中兴通讯，Rubric ③）
10. 闭环矩阵与 TN 水分（Rubric ④）
11. 失败案例（Rubric ④）
12. Vibe Coding 反思（Rubric ⑤）
13. 局限与下一步
14. 答辩问题预案

## 不得写入内容

- ❌ overclaim 措辞（与 M01 约束一致）
- ❌ 编造图表路径（所有来源必须指向真实文件/字段）
- ❌ 跳过 §12.20 任何检查项
