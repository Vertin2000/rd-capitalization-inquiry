# 难度档位声明

> Week 11 导论 2.10 / Week 12 4.7 要求：小组声明难度档位，Week 15 可调整但需说明理由。

## 声明档位

**挑战档 1.1**

## 选择理由

本项目在课程基础要求之上叠加以下挑战档特征（对应 Week 11 导论 2.9 挑战分 Rubric）：

### 数据规模与质量（目标 8 分）

- **年报 PDF**：150 份（50 家研发密集型上市公司 × 2021-2023 年），超过挑战档「150+ 份」门槛。
- **核心字段**：14 个 Schema 字段，超过挑战档「10 个以上」门槛。
- **MinerU 解析**：150/150 通过 api-batch 全量解析，275 个 page segment 全部 success。

### 任务复杂度（目标 8 分）

- **多公告类型**：年报主线 + 问询函/回复函/关注函辅助公告匹配。
- **多阶段事件**：company-year 跨期变化（2021-2023 资本化率 Z 评分）。
- **闭环**：全样本问询候选发现 → 下载 → 标签 → 混淆矩阵，覆盖全部 150 个 company-year（不止查异常样本）。
- **评分**：四维度风险评分模型（行业偏离 / 跨期变化 / 条件模糊度 / 恒等式置信度）。

### 自主性与创新性（目标 4 分）

- 确定性表格提取（`src/extract/rd_table_extractor.py`）作为 LLM fallback，提升数值字段可靠性。
- 问询相关性 v2 标签器：脚本剪枝 + Tier-1 关键词 + LLM 语义二分类，非简单关键词 OR。
- 官方 API 路线（OAuth2 + `p_info3015`）替代前端 AJAX，doc_id 人可读化。

## 档位匹配说明

- 选题来自 Week 11 选题池参考（研发资本化 + 问询函闭环），属挑战档特征，非基础档/标准档。
- 数据量不是唯一标准：即便部分样本字段缺失（67 条 partial），选题的多公告匹配、跨期追踪、评分与闭环特征仍满足挑战档定义。
- Week 15 起不再调整档位。

```
项目二得分 = min(100, 基础质量分 80 + 挑战分 20 × 难度系数 1.1)
```
